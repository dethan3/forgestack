# 与 AI 协作指南

## 核心原则

```
AI 是工具，你是决策者。
AI 生成代码，你负责审核。
AI 提供方案，你做出选择。
```

## 有效的 Prompt 技巧

### 1. 描述清楚需求

❌ 模糊的请求：
```
帮我写一个函数
```

✅ 清晰的请求：
```
帮我写一个 JavaScript 函数：
- 功能：计算数组中所有正数的平均值
- 输入：数字数组
- 输出：平均值（如果没有正数返回 0）
- 要求：使用 ES6+ 语法
```

### 2. 提供上下文

❌ 缺少上下文：
```
这段代码报错了，帮我修复
```

✅ 提供完整信息：
```
这段代码报错了：

代码：
[粘贴代码]

错误信息：
[粘贴错误]

期望行为：
[描述期望]
```

### 3. 指定约束条件

```
请帮我实现一个去重函数：
- 使用 ES6+ 语法
- 不能使用 Set
- 保持原有顺序
- 时间复杂度尽量优化
```

### 4. 要求解释

```
请实现并解释：
1. 代码实现
2. 为什么这样写
3. 有哪些边界情况需要注意
4. 有没有其他实现方式
```

## 审核 AI 代码清单

生成代码后，检查以下要点：

### 功能正确性
- [ ] 是否满足所有需求
- [ ] 边界情况是否处理
- [ ] 输入输出是否正确

### 代码质量
- [ ] 变量命名是否清晰
- [ ] 是否有不必要的代码
- [ ] 是否符合现有代码风格

### 安全性
- [ ] 是否有 XSS 风险
- [ ] 是否有注入风险
- [ ] 敏感数据是否暴露

### 性能
- [ ] 是否有明显性能问题
- [ ] 是否有内存泄漏风险
- [ ] 复杂度是否合理

### 可维护性
- [ ] 代码是否易于理解
- [ ] 是否方便后续修改
- [ ] 是否有适当的注释

## 常用 Prompt 模板

### 学习概念
```
请解释 JavaScript 中的 [概念名称]：
1. 是什么
2. 为什么需要它
3. 什么时候使用
4. 给一个简单示例
```

### 代码审核
```
请审核这段代码：

[粘贴代码]

检查：
1. 是否有 bug
2. 是否有安全问题
3. 代码风格是否良好
4. 有什么改进建议
```

### 重构代码
```
请帮我重构这段代码：

[粘贴代码]

要求：
1. 使用 ES6+ 语法
2. 提高可读性
3. 保持功能不变
4. 解释修改的原因
```

### 调试错误
```
我遇到了一个错误：

代码：
[粘贴代码]

错误：
[粘贴错误信息]

我尝试过：
[描述尝试的解决方法]

请帮我：
1. 分析错误原因
2. 提供解决方案
3. 解释如何避免类似问题
```

### 比较方案
```
我有两种实现方案：

方案 A：
[代码 A]

方案 B：
[代码 B]

请比较：
1. 各自的优缺点
2. 性能差异
3. 适用场景
4. 你的推荐
```

## 警惕事项

### AI 可能出错的地方

1. **最新 API** - AI 知识可能过时
2. **边界情况** - 可能遗漏特殊情况
3. **上下文理解** - 可能误解你的需求
4. **最佳实践** - 可能给出次优方案

### 不要盲目信任

- 始终运行测试验证
- 仔细阅读生成的代码
- 不懂的地方要追问
- 与文档/资料交叉验证

## 成为更好的 AI 协作者

```
初级：让 AI 写代码 → 直接使用
中级：让 AI 写代码 → 审核后使用
高级：让 AI 写代码 → 审核 → 改进 → 理解原理
```
